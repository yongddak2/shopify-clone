01회원 / 인증

JWT(access 30분 + refresh 7일) 기반. 401 시 axios 인스턴스에서 자동 재발급.

회원가입 USER

  • 이메일/비밀번호/이름/전화번호
  • 약관 동의 3종 (이용약관 / 개인정보 / 마케팅)
  • 비밀번호 정책: 8자+영문+숫자+특수문자
  • 가입 시 웰컴 쿠폰 자동 발급 (실패해도 가입 진행)
frontend/src/app/signup/page.tsx · AuthController

로그인 / 로그아웃 USER

  • access / refresh 토큰 발급, refresh는 Redis 보관
  • 탈퇴 회원 차단 (findByEmailAndDeletedAtIsNull)
  • 소셜 로그인 provider 컬럼 보유 (LOCAL/KAKAO/GOOGLE)
frontend/src/app/login/page.tsx · stores/authStore.ts

비밀번호 찾기 USER

  • 3단계 흐름: 코드 발송 → 인증 → 새 비밀번호
  • Redis 코드 3분 · verified 키 10분 · 재발송 30초
  • 소셜 로그인 계정은 차단
frontend/src/app/forgot-password/page.tsx · PasswordResetController

회원 정보 / 비밀번호 변경 USER

  • 이름·전화번호 수정
  • 비밀번호 변경 30일 제한 (passwordChangedAt)
  • 주소록 CRUD + 기본 배송지 지정
mypage/profile · mypage/addresses · UserController

04주문 / 결제

토스페이먼츠 + 비관적 락(PESSIMISTIC_WRITE) 기반 재고 정합성.

주문 생성→ 토스 requestPayment→ /payment/success→ POST /api/payments/confirm→ 쿠폰 markUsed→ invalidateAfterPayment

주문 생성 USER

  • 카카오 주소 API + 쿠폰 적용
  • 배송비: 50,000원 이상 무료 / 미만 3,000원
  • 재고 락 후 decreaseStock + 락 객체 refresh
  • 주문번호: ORD-{millis}-{UUID앞8}
order/page.tsx · OrderService.createOrder

결제 승인 USER

  • 토스 API: orderNumber 기반 confirm
  • PAID 시 판매량(salesCount) increase
  • PAID 시 쿠폰 markUsed() + 저장 필수
  • 주문 확인 이메일 자동 발송 (@Async)
PaymentService · payment/success/page.tsx

주문 취소 USER

  • PENDING / PAID 에서만 가능

  • 다중 옵션 시 optionValueId 오름차순 락 (데드락 방지)

11이메일 알림

Gmail SMTP + @Async. OrderEmailContext 스냅샷으로 LazyInitializationException 회피.

PAID 알림

  • 결제 완료 시 자동 발송
  • 주문 번호 · 결제 금액 · 배송지

SHIPPED 알림

  • 관리자가 SHIPPED 변경 시 발송
  • 실제 운송장 번호 + 배송사 포함
  • null 이면 "준비 중" fallback

CANCELLED 알림

  • 관리자 취소만 발송
  • 사용자 직접 취소는 미발송

비밀번호 코드

  • 비밀번호 찾기 인증 코드
  • Redis 3분 만료
